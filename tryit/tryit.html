These fields define the puzzle to apply move sequences to.

Shape

This field defines the shape of the puzzle (cube, dodecahedron, pyramid, ...). For now, only the cube shape is supported.

Size

This field defines the number of layers of the puzzle.

Stage

This field defines the stage at which the puzzle must be displayed (OLL, PLL, F2B, ...). This option is not yet supported.

Color scheme

This field defines for each face, the color of all stickers of the face.


Remark : In the input object required by Holo-Cube, the puzzle is described by the single property "fullName". On this page the shape and size of the puzzle are artificially separated to ease the understanding.

This field defines the move sequences to apply to solved puzzles.


One move sequence per line : move sequences must be separated by a newline character.


In the move sequences, moves must be separated by a space.

These fields define some options for drawing the images. The default values are :

  • View : "plan"
  • Image height = 100px, image width = 100px, image scale = 1
  • Image background color : transparent
  • Puzzle height = 100px, puzzle width = 100px, puzzle scale = 1
  • Puzzle color : black

View

This defines the view mode of the puzzle :

  • Plan : view from above, showing the U face and the adjacent stickers
  • Isometric : view from 45° above and 45° aside, showing the up, front and right sides
  • Top-down : view from 45° above, showing the up and front sides
  • Net : net representation showing all stickers

Image height and width

These fields define the height and width of the image, in pixels.

Image scale

This field defines a multiplier to apply to both image dimensions. If some of the image dimensions are not specified, the multiplier will be applied after taking the respective default values.

Image background color

This field defines the background color of the image.

Puzzle height and width

These fields define the height and width of the puzzle, in pixels.

Puzzle scale

This field defines a multiplier to apply to both puzzle dimensions. If some of the puzzle dimensions are not specified, the multiplier will be applied after taking the respective default values.
After the multiplier is applied, both puzzle dimensions must not exceed the image dimensions.

Puzzle color

This field defines the color of the puzzle between the stickers (= plastic color).

During its execution, Holo-Cube writes some logs to help following the different steps. These fields define how much logs and where these logs will be written.

Mode

This field defines where the logs will be written.

  • Console (default) : Logs are written in the javascript console.
  • HTML tag : Logs are written in the HTML document, inside the tag specified by the selector (same as CSS or document.querySelector()). This option is only available in web pages.
  • Off : No logs are written.

Remark : Even if the selected mode is not "Console", the errors will also be displayed in the console. This way debug is easier when trying to use another log mode (for example if the HTML selector is invalid).

In output

This field defines if the logs should also be included in the result object, under the "log" node at root level. These logs include the logs of initialization phase, the logs of execution phase, and both combined.

Remark : If the execution handles a list of move sequences instead of a single move sequence, the logs of execution phase are also written in the result of each move sequence, in addition to the logs at root level.

Verbosity

This field defines how much logs will be written. Each level of verbosity n will include all logs of level n-1.

  • -1 : Logs are fully disabled (not recommended), for both console/htmlTag modes and in output.
  • 0 : Only errors will be displayed.
  • 1 (default): In addition to 0, some general logs are written.
  • 2 : In addition to 1, some more detailed logs are written.
  • 3 : In addition to 2, some debug logs are written.
